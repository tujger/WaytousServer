Rest calls

Regular request: https://domain.com/PATH?OPTIONS.

Example: https://domain.com/rest/api?version

Regular response: 

    {
        request: String,
        code: Int,
        extra: Object,
        message: Object,
        status: String
    }
Here:
  • Request - copy of requested path
  • Code means what the message contains:
    1 - String; if status=error then means "Action not defined",
    2 - List; if status=error then means "Runtime error",
    3 - JSON,
    4 - HTML,
    302 - redirect, then message contains redirect link,
    400 and above - error codes (and also status=error)
  • Extra (see details)
  • Message (see details)
  • Status:
    success
    error
